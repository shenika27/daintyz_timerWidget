# 타이머 영역 렌더링 / 제약 (개발용)

> **스킨 제작 방법**(캔버스 크기·프레임·바닥 기준선·애니메이션 속도·배포 절차)은
> 스킨 리포(`daintyz_timer_characterList`)의 README를 참고할 것.
> 이 문서는 **위젯 코드가 타이머 영역을 어떻게 렌더링하는지와 RemoteViews 제약**을 기록한다.

핵심 원칙: **"기능 = 영역, 버튼 = 그림"** — 조작은 항상 존재하는 투명 탭 영역이 담당하고,
보이는 박스/선/버튼/숫자 글꼴은 모두 스킨이 "그린다".

**가로 레이아웃(2024 재작업)**: 타이머 영역은 **왼쪽 시간(~80%) + 오른쪽 버튼 2개 세로 컬럼(~20%)**.
버튼이 항상 2개라 모든 상태에서 선 구성(시간↔버튼 세로선 1 + 버튼 사이 가로선 1)이 동일하다.
→ 그래서 **박스·구분선을 모두 스킨의 타이머 배경 PNG(`background`)에 구워 넣고**, 코드의 구분선 시스템은 제거했다.

관련 파일:
- 레이아웃: `app/src/main/res/layout/widget_section_timer.xml` (가로 배치)
- 래퍼/크기: `widget_timer_top.xml`·`widget_timer_bottom.xml`(타이머/캐릭터 비율), `res/xml/widget_info.xml`(2x2)
- 스타일: `app/src/main/res/values/styles.xml` (`WidgetTimerButton`)
- 렌더링: `app/src/main/java/com/daintyz/timerwidget/widget/WidgetUpdater.kt`
- 스킨 스키마: `app/src/main/java/com/daintyz/timerwidget/model/SkinModel.kt` (`TimerSkin`, `TimerFont`)

## 1. 영역 구조 (가로)

```
┌───────────────────────────┬─────┐  ← iv_timer_bg (박스·선이 구워진 스킨 배경 PNG, fitXY)
│                           │  +  │  ← 우측 버튼 컬럼(세로): 위/아래 2개만 VISIBLE
│         10:00             ├─────┤     (선은 배경 PNG에 포함)
│   (tv_timer_value, 탭)    │  −  │
└───────────────────────────┴─────┘
        weight 4 (~80%)      weight 1 (~20%)
```

- 시간(`tv_timer_value`)은 **세로 가운데** 정렬, 영역 전체가 시작/일시정지 탭 타깃.
- 버튼 컬럼 자식(위→아래) = `btn_plus`, `btn_minus`, `btn_start_pause`, `btn_stop_reset`.
  GONE 접힘으로 **항상 2개만** 보이고, 보이는 2개가 컬럼 높이를 균등 2분할한다.
- 시간↔버튼 사이 세로선, 두 버튼 사이 가로선은 **코드가 그리지 않는다** — 배경 PNG에 그려져 있다.

### 상태별 구성 (`WidgetUpdater.applyButtonVisibility` / `applyButtonActions`)

| 상태 | 위 버튼 | 아래 버튼 | 시간 표시 | 시간 탭 동작 |
|------|--------|----------|----------|-------------|
| IDLE(정지) | `+` | `−` | 설정시간 | 시작(`ACTION_START_PAUSE`) |
| RUNNING(진행) | `‖` 일시정지 | `■` 초기화 | 남은시간 | 일시정지 |
| PAUSED(중지) | `▶` 계속 | `■` 초기화 | 남은시간 | 재개 |
| COMPLETE(완료) | `+` | `−` | `00:00` | 이전 설정시간으로 초기화(`ACTION_TAP_COMPLETE`) |

- 우측 상단 버튼은 `btn_start_pause` 한 슬롯이 상태에 따라 `play`(PAUSED) ↔ `pause`(RUNNING)로 바뀐다.
- **시간 탭과 우측 버튼은 둘 다 동작한다**(중복 허용 — 시간 영역은 큰 탭 타깃). 완료는 IDLE과 화면이
  같고 시간 탭 동작만 다르다(별도 "완료" 오버레이 없음).
- 버튼이 GONE이어도 탭 영역 자체가 사라지는 것이지, 노스킨에서 "그림만" 투명한 것과는 다르다.

## 2. 위젯 크기 / 비율

- 위젯은 **2x2** 고정(`widget_info.xml`: `minWidth/minHeight = 110dp`, `targetCellWidth/Height = 2`).
- 한 칸 안에서 **타이머 ~30%(`layout_weight=3`) / 캐릭터 ~70%(`layout_weight=7`)** + 사이 4dp 여백.
  비율은 `widget_timer_top.xml`·`widget_timer_bottom.xml`의 `<include>` weight로 조정한다.
  (include로 weight를 적용하려면 `layout_width`+`layout_height`를 둘 다 명시해야 한다.)

### 기준 치수

| 요소 | 값 |
|------|-----|
| 타이머 박스 바깥 마진 | 좌우 6dp (include `layout_marginHorizontal`) |
| 박스 코너 라운드 / 테두리 (무배경 폴백) | 10dp / 3dp (`timer_box_bg`) |
| 숫자 폰트 | monospace · 30sp · bold · `timer_digit` |
| 시간/버튼 가로 비율 | weight 4 : 1 (~80% / ~20%) |
| 버튼(ImageView) | 폭=`match_parent` × 높이=weight 1(컬럼 2분할), padding 5dp, `fitCenter` |

> 시간/버튼 비율(4:1)을 바꾸면 **배경 PNG의 세로선 위치도 같이 맞춰야** 한다(아래 디자인 계약 참고).

## 3. 스킨 스키마 (`skin.json` 의 `timer` 블록)

```json
"timer": {
  "background": "timer_theme.png", // 박스·세로선·버튼사이 가로선이 모두 그려진 타이머 배경. 출시 테마는 필수.
  "showBox": true,           // background가 없을 때만 쓰는 내장 박스 폴백(개발/테스트용). false면 투명.
  "buttonStyle": "skin",     // "default"=내장 기호 벡터 / "none"=그림 없음 / "skin"=스킨 PNG
  "buttons": {               // buttonStyle="skin"일 때만 사용 (그 외 무시)
    "minus": "btn_minus.png",
    "plus":  "btn_plus.png",
    "play":  "btn_play.png",
    "pause": "btn_pause.png",
    "stop":  "btn_stop.png"
  },
  "font": {
    "family": "monospace",   // 내장 패밀리명 (※ 4절 제약 참고)
    "color": "#1A1A1A",      // "#RRGGBB" 또는 "#AARRGGBB"
    "sizeSp": 30
  }
}
```

- `timer` 블록이 **없으면 = 기본 스킨**: 빈 JSON으로 파싱돼 내장 박스/기호 버튼이 적용된다
  (`SkinRepository.parseSkinJson` 의 `optJSONObject("timer") ?: JSONObject()`).
- 과거 divider 필드(`showDividerH`/`dividerHImage`/`dividerHHeightDp`/`showDividers`/`dividersColor`/
  `dividersImage`/`dividersWidthDp`)는 **모두 제거**됐다. 기존 skin.json에 남아 있어도 무시된다(하위호환).
- `font`의 각 필드는 생략 가능 — 생략 시 레이아웃 기본값 유지.

### 타이머 배경 (`background`) — 디자인 계약

배경 PNG는 타이머 영역 전체에 `fitXY`로 깔리고, 그 위에 숫자(왼쪽)·버튼 아이콘(오른쪽)이 겹쳐 그려진다.
**박스 테두리 + 시간↔버튼 세로선 + 버튼사이 가로선을 모두 이 한 장에 그린다.**

```
┌───────────────────────────┬─────┐
│                           │  ▒  │   세로선 = 4:1 비율 경계(폭의 약 80% 지점)
│         (숫자 자리)        │▒▒▒▒▒│   가로선 = 버튼 컬럼 세로 중앙
│                           │  ▒  │
└───────────────────────────┴─────┘
```

- **세로선은 폭의 ~80% 지점**(weight 4:1 경계)에 그린다. 비율을 바꾸면 이 위치도 같이 바꿔야 한다.
- **가로선은 우측 버튼 컬럼의 세로 중앙**에 그린다(두 버튼을 나누는 선).
- 좌우/상하 끝까지 이어지는 단순·대칭 구성을 권장(`fitXY`로 늘어나므로 코너 장식은 왜곡됨).
- 로드 실패 시 내장 박스(`showBox`)로 폴백하되, 출시 테마는 배경을 무조건 포함한다.

### 버튼 그림 방식 (`buttonStyle`)

| 값 | 동작 |
|----|------|
| `"default"` | 내장 벡터(`ic_btn_*`)를 그린다. (현재 potato/pang) |
| `"none"` | 그림 없음(투명). 탭 영역만 유지. |
| `"skin"` | `buttons`에 지정한 스킨 PNG를 그린다. **벡터는 숨김.** |

- **심볼 단위 5장**(`minus`/`plus`/`play`/`pause`/`stop`). 버튼 슬롯은 4개지만 `start_pause` 슬롯이
  상태에 따라 `play`(PAUSED) ↔ `pause`(RUNNING)로 바뀌고, `stop`은 RUNNING·PAUSED에서 재사용된다.
- `buttons`에서 **누락한 심볼은 자동으로 내장 벡터로 폴백** → 일부만 교체도 가능.
- PNG는 버튼 칸(폭=`match_parent` × 높이=컬럼 2분할, padding 5dp)에 `fitCenter`로 들어간다.
- **버튼 PNG는 반드시 투명 배경(RGBA)** — `background` 위에 겹쳐 그려지므로 불투명하면 배경을 가린다.

### zip 구성 (출시 테마)

상황별 캐릭터(`idle_01`, `run_01`, `stop_01`…) + 버튼 5종 PNG + `timer_theme` 배경 1장.
선/박스를 따로 내려줄 필요가 없어 구성이 단순해졌다.
(디자인 레포 결합: 카탈로그는 raw, 에셋은 jsDelivr 캐시 규칙을 따른다.)

## 4. RemoteViews 제약 (위젯이라 반드시 지킬 것)

위젯은 RemoteViews라 일반 View와 다르게 **제약**이 있다. 어기면 런처에 **"Can't load widget"**.

1. **순수 `<View>` 태그 금지** — 인플레이션 화이트리스트에 없음. 칸막이는 `TextView`/`ImageView`/`Space`로.
2. **비-remotable 메서드 호출 금지** — `setString/setInt` 등 리플렉션 호출은 `@RemotableViewMethod`만 가능.
   - 가능: `setTextColor`, `setTextViewTextSize` → **글자 색/크기는 직접 지정 OK**.
   - 불가: `setFontFamily(String)` → **글꼴(family)·커스텀 .ttf는 직접 불가**.
     커스텀 글꼴은 숫자를 `Typeface`로 **비트맵 렌더링 후 `setImageViewBitmap`** 방식으로만 가능(향후 확장 지점).
3. 비트맵은 IPC 크기 제한이 있으니 버튼/배경 PNG는 표시 크기에 맞게(과대 해상도 X).

진단: 위젯이 깨지면 에뮬레이터 재설치 후 `adb logcat | grep AppWidgetHostView` 로 정확한 예외/줄번호 확인.

## 5. "희끄무리한" 배경(런처 위젯 패널)

위젯 콘텐츠가 셀을 꽉 채우지 않고 투명 영역이 크면, 일부 런처(예: 삼성 One UI 등)는
위젯 뒤에 **옅은 둥근 배경 패널**을 깔아준다. 이는 타이머 스킨/크기 문제가 아니라 **런처 동작**이다.
(Pixel 런처 기준 테스트기에서는 타이머 주변이 깨끗하게 렌더링됨.)

완화 방법:
- 스킨이 영역 배경을 불투명/반투명으로 채워 "패널처럼" 보이게 디자인.
- 또는 그대로 두고 투명을 유지(런처별 차이로 수용).

> 캐릭터 프레임의 바닥 기준선 정렬, 캔버스 크기, 애니메이션 속도(`frameDurationMs`) 등
> **그림 제작 규칙은 스킨 리포 README로 이관**되었다.

## 6. 노스킨 규칙 요약

- 배경 투명, 박스 없음, 버튼 그림 없음. **`timer` 블록에 `showBox:false`, `buttonStyle:"none"`을 명시**해야 한다.
  (※ `timer` 블록 자체를 생략하면 노스킨이 아니라 **기본 스킨**이 적용된다.)
- 단, **버튼 탭 영역(ImageView)과 시간 탭은 유지** → 조작은 그대로 가능.
- 숫자 글꼴은 레이아웃 기본값. (어두운 배경에서 가독성이 필요하면 스킨에서 `font.color` 지정 권장.)
