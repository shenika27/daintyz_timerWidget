# 타이머 영역 렌더링 / 제약 (개발용)

> **스킨 제작 방법**(캔버스 크기·프레임·바닥 기준선·애니메이션 속도·배포 절차)은
> 스킨 리포(`daintyz_timer_characterList`)의 README를 참고할 것.
> 이 문서는 **위젯 코드가 타이머 영역을 어떻게 렌더링하는지와 RemoteViews 제약**을 기록한다.

핵심 원칙: **"기능 = 영역, 버튼 = 그림"** — 조작은 항상 존재하는 투명 탭 영역이 담당하고,
보이는 박스/구분선/버튼/숫자 글꼴은 모두 스킨이 "그린다".

관련 파일:
- 레이아웃: `app/src/main/res/layout/widget_section_timer.xml`
- 스타일: `app/src/main/res/values/styles.xml` (`WidgetTimerButton`, `WidgetButtonDivider`)
- 렌더링: `app/src/main/java/com/daintyz/timerwidget/widget/WidgetUpdater.kt`
- 스킨 스키마: `app/src/main/java/com/daintyz/timerwidget/model/SkinModel.kt` (`TimerSkin`, `TimerFont`)

## 1. 영역 구조

타이머 영역은 세로로 3단:

```
┌────────────────────────────┐  ← timer_box (배경/테두리 = 스킨)
│           10:00            │  ← tv_timer_value (숫자, 폰트 = 스킨)
├────────────────────────────┤  ← div_h  (가로 구분선, 박스 양끝까지)
│    −    |    ▶    |    +  │  ← 버튼 행 (버튼 = 그림, | = 세로 구분선)
└────────────────────────────┘
```

- **가로 구분선(`div_h`)**: 좌우 마진 없이 박스 양 끝(테두리)에 붙는다.
- **세로 구분선(`div_minus`/`div_plus`/`div_stop`)**: 높이 `match_parent` — 위는 가로선, 아래는 박스 하단에 붙는다.
- 세로선은 옆 버튼이 보일 때만 표시(상태 연동) + 스킨의 `showDividers`가 켜져 있어야 표시.

### 상태별 버튼 노출
| 상태 | 보이는 버튼 |
|------|-------------|
| IDLE(정지) | `−  ▶  +` |
| RUNNING(진행) | `▶(=II)  □` |
| PAUSED(일시정지) | `▶  □` |
| COMPLETE(완료) | 버튼 행(`btn_row`) 전체 GONE, 대신 `tv_complete`("완료") 표시 |

버튼이 GONE이어도 **탭 영역 자체가 사라지는 것이지**, 노스킨에서 "그림만" 투명한 것과는 다르다.

**완료 상태 처리** (`WidgetUpdater`):
- 숫자(`tv_timer_value`)는 특수 텍스트 없이 `00:00`(남은 시간 0).
- 버튼 행(`btn_row`)을 통째로 GONE, 같은 자리의 `tv_complete`("완료")를 VISIBLE로.
- `tv_complete` 글자색은 `timer.font.color`를 따라가고(숫자와 동일), 크기는 자체 16sp.
- `tv_complete` 탭 → `ACTION_TAP_COMPLETE` → `resetFromComplete`(IDLE 복귀, 직전 설정 시간 유지). 캐릭터 탭과 동일.

## 2. 기준 치수 (디폴트 스킨 = potato 현재값)

| 요소 | 값 |
|------|-----|
| 박스 바깥 마진 | 좌우 6dp / 상하 2dp |
| 박스 코너 라운드 | 10dp (`timer_box_bg`) |
| 박스 테두리 | 3dp (`timer_box_border`) |
| 숫자 폰트 | monospace · 30sp · bold · `#1A1A1A` |
| 숫자 좌우 패딩 | 8dp (박스가 아닌 숫자에만 — 가로선이 끝까지 닿도록) |
| 버튼(ImageView) | 폭=weight 1(칸 균등 분할) × 높이 32dp, padding 5dp, `scaleType=fitCenter` |
| 가로 구분선 | 높이 1dp, 박스 폭 꽉 |
| 세로 구분선 | 폭 1dp, 높이 `match_parent` |

> 박스 상하 마진을 키우면 캐릭터와의 간격이 벌어진다. 현재 2dp는 간격 최소화 값.

## 3. 스킨 스키마 (`skin.json` 의 `timer` 블록)

```json
"timer": {
  "background": "timer_bg.png", // 스킨이 그린 박스 배경 PNG(스킨 폴더 기준). 생략 시 내장 박스(showBox)
  "showBox": true,          // 박스 배경/테두리 표시 (false면 투명, background 있으면 무시)
  "showDividerH": true,     // 숫자/버튼 사이 가로선. 배경 이미지에 그려넣으면 false
  "dividerHImage": "divider_h.png", // 가로 구분선 이미지(스킨 폴더 기준). 지정 시 색 대신 PNG
  "dividerHHeightDp": 1,    // 가로 구분선 높이(dp). dividerHImage 사용 시 필수 조정. 기본값 1dp
  "showDividers": true,     // 버튼 사이 세로선. 버튼 수가 상태마다 달라 배경 이미지로 대체 불가
  "dividersColor": "#2B2B2B", // 세로 구분선 색. 생략 시 기본값(#2B2B2B). dividersImage 있으면 무시
  "dividersImage": "divider.png", // 세로 구분선 이미지(스킨 폴더 기준). 지정 시 색 대신 PNG 사용
  "dividersWidthDp": 1,       // 세로 구분선 폭(dp). dividersImage 사용 시 필수 조정. 기본값 1dp
  "buttonStyle": "default", // "default"=내장 기호 벡터 / "none"=그림 없음 / "skin"=스킨 PNG
  "buttons": {              // buttonStyle="skin"일 때만 사용 (그 외 무시)
    "minus": "btn_minus.png",
    "plus":  "btn_plus.png",
    "play":  "btn_play.png",
    "pause": "btn_pause.png",
    "stop":  "btn_stop.png"
  },
  "font": {
    "family": "monospace",  // 내장 패밀리명 (※ 4절 제약 참고)
    "color": "#1A1A1A",     // "#RRGGBB" 또는 "#AARRGGBB"
    "sizeSp": 30
  }
}
```

- `timer` 블록이 **없으면 = 기본 스킨**: 빈 JSON으로 파싱돼 내장 박스/구분선/기호 버튼이 적용된다
  (`SkinRepository.parseSkinJson` 의 `optJSONObject("timer") ?: JSONObject()`). 노스킨은 아래처럼 명시해야 한다.
- `font`의 각 필드는 생략 가능 — 생략 시 레이아웃 기본값 유지.

### 타이머 박스 배경 (`background`)

배경 PNG는 박스 전체 영역에 `fitXY`(폭·높이 꽉 채워 늘림)로 깔리고, 그 위에 숫자·구분선·버튼 아이콘이 그려진다.
지정되면 내장 박스 모양(`showBox`)은 자동으로 꺼진다. 로드 실패하면 내장 박스로 폴백.

#### 영역 구조 및 치수

```
┌──────────────────────────────────────┐  ← 박스 시작 (좌우 마진 6dp / 상하 2dp는 박스 바깥)
│              10 : 00                 │  ← 숫자 영역: 30sp bold, 상하 padding 5dp
├──────────────────────────────────────┤  ← 가로 구분선 1dp (showDividers)
│    [  −  ] │ [  ▶  ] │ [  +  ]      │  ← 버튼 행: 높이 32dp, 각 칸 1/3 균등 (IDLE)
│    [  ‖  ] │ [  ■  ]                │  ← 버튼 행: 높이 32dp, 각 칸 1/2 균등 (RUNNING/PAUSED)
└──────────────────────────────────────┘
```

#### 버튼 칸 위치는 상태에 따라 바뀐다

버튼은 `weight=1` 균등 분할이라, **보이는 버튼 수에 따라 각 칸의 폭이 달라진다.**

| 상태 | 보이는 버튼 | 칸 구성 |
|---|---|---|
| IDLE | − ▶ + | 폭 1/3씩 3등분 |
| RUNNING | ‖ ■ | 폭 1/2씩 2등분 |
| PAUSED | ▶ ■ | 폭 1/2씩 2등분 |
| COMPLETE | "완료" 텍스트 | 버튼 행 대신 `tv_complete` 한 줄(행 전체) |

→ 배경 이미지에 버튼 칸 구분을 표시하면 상태에 따라 위치가 맞지 않는다.
**배경에는 칸 구분선을 그리지 말 것.** 아이콘 위치는 코드가 자동 배치하고, 구분선이 필요하면 `showDividers: true`를 쓴다(단, 위의 주의사항 참고).

#### 세로 비율 가이드

배경 이미지의 세로는 **숫자 영역 + 구분선 1dp + 버튼 행 32dp** 로 결정된다.
숫자 영역 높이는 폰트 크기(기본 30sp)에 따라 유동적이므로, 이미지 비율보다 **상단(숫자)과 하단(버튼) 영역을 색/명도로 구분**해 두는 디자인이 늘어남(stretch)에 강하다.

#### 가로 늘어남 대응

박스 폭은 위젯 셀 크기에 따라 달라진다. 코너 장식이 있으면 `fitXY` 특성상 왜곡되므로 **단순·대칭 구성**을 권장한다.

### 구분선 표시 (`showDividerH` / `showDividers`)

구분선을 가로/세로로 분리해 제어한다.

| 필드 | 대상 | 설명 |
|---|---|---|
| `showDividerH` | 가로선(숫자↔버튼 사이) | 배경 이미지에 그려넣을 수 있어 선택적 |
| `showDividers` | 세로선(버튼 사이) | 버튼 수가 상태마다 달라 배경 이미지로 대체 불가 |

두 선 모두 `background` 이미지 **위에** 겹쳐 그려진다.

**유료 스킨 권장 조합:**

| 항목 | 권장값 | 이유 |
|---|---|---|
| `showDividerH` | `false` | 가로선은 배경 이미지에 포함 |
| `showDividers` | `true` | 세로선은 코드가 상태에 맞춰 자동 배치 |
| `dividersColor` | 디자인에 맞는 색 지정 | 세로선 색을 배경과 어울리게 |

### 가로 구분선 이미지 (`dividerHImage` / `dividerHHeightDp`)

세로선과 동일한 방식. `showDividerH: true`일 때만 적용된다.

| 필드 | 설명 |
|---|---|
| `dividerHImage` | PNG 파일명. 지정 시 `fitXY`로 가로선 영역에 채움. 없으면 단색(`#2B2B2B`) |
| `dividerHHeightDp` | 가로선 높이(dp). **이미지 사용 시 필수.** 생략 시 1dp — 이미지가 1dp에 압축돼 보이지 않음 |

가로선은 박스 **전체 폭**을 차지하므로, 이미지는 가로로 길고 세로는 얇은 형태(예: 8×2px RGBA → `fitXY`로 늘림)가 적합하다.
좌우 끝이 박스 테두리와 맞닿으므로, 디자인이 양쪽 끝까지 이어지는 형태로 제작할 것.

```json
"showDividerH": true,
"dividerHImage": "divider_h.png",
"dividerHHeightDp": 8
```

### 세로 구분선 이미지 (`dividersImage` / `dividersWidthDp`)

단색으로 표현하기 어려운 구분선 디자인(그라데이션, 기둥+식물 등)은 PNG 이미지로 교체할 수 있다.

#### 동작 방식

- `dividersImage` 지정 → PNG를 `fitXY`(폭·높이 꽉 채워 늘림)로 구분선 영역에 그린다.
- `dividersImage` 없음 → `dividersColor`(또는 기본 `#2B2B2B`) 단색 선.
- 이미지와 색은 **동시에 적용되지 않는다** — 이미지가 있으면 색은 무시된다.

#### `dividersWidthDp` 상세

세로 구분선의 폭(dp)을 런타임에 결정한다.

| 값 | 결과 |
|---|---|
| 생략(기본 1dp) | 1dp 단색 선. 색 모드에서만 의미있음 |
| 이미지 사용 시 1dp | 이미지가 1dp 폭에 압축돼 사실상 보이지 않음 → **반드시 넓혀야 함** |
| 12~20dp | 기둥/식물 등 장식 구분선에 적합한 범위 |

> **이미지 사용 시 `dividersWidthDp`를 생략하면 구분선이 보이지 않는다.** 반드시 함께 지정할 것.

#### 버튼 칸 폭에 미치는 영향

버튼은 `weight=1` 균등 분할이라, 구분선 폭만큼 버튼 영역이 줄어든다.

```
전체 박스 폭 W, 구분선 폭 D 일 때:

IDLE     (버튼 3 / 구분선 2개): 버튼 칸 폭 = (W - D×2) / 3
RUNNING  (버튼 2 / 구분선 1개): 버튼 칸 폭 = (W - D×1) / 2
```

→ `dividersWidthDp`가 커질수록 버튼 칸이 좁아진다. 아이콘이 잘리지 않게 적정 폭을 유지할 것.
→ 같은 D 값이라도 IDLE(구분선 2개)과 RUNNING(구분선 1개)에서 버튼 칸 폭이 달라진다.
  **구분선 이미지는 이 차이를 감안해 좌우 여백 없이 단순한 형태**로 디자인하는 것이 자연스럽다.

#### 예시

```json
"showDividers": true,
"dividersImage": "divider.png",
"dividersWidthDp": 16
```

`divider.png`: 세로로 긴 이미지(예: 16×64px RGBA). `fitXY`로 구분선 높이에 맞춰 늘어난다.

### 버튼 그림 방식 (`buttonStyle`)
| 값 | 동작 |
|----|------|
| `"default"` | 내장 벡터(`ic_btn_*`)를 그린다. (현재 potato/pang) |
| `"none"` | 그림 없음(투명). 탭 영역만 유지. |
| `"skin"` | `buttons`에 지정한 스킨 PNG를 그린다. **벡터는 숨김.** |

- **심볼 단위 5장**(`minus`/`plus`/`play`/`pause`/`stop`)을 그린다 — 버튼 슬롯은 4개지만
  `start_pause` 슬롯이 상태에 따라 `play`(IDLE·PAUSED) ↔ `pause`(RUNNING)로 바뀌기 때문.
  `stop`은 RUNNING·PAUSED, `play`는 IDLE·PAUSED에서 재사용된다.
- `buttons`에서 **누락한 심볼은 자동으로 내장 벡터로 폴백** → 일부만 교체도 가능.
- PNG는 버튼 칸(폭=weight 균등분할 × 높이 32dp, padding 5dp)에 `fitCenter`로 들어간다. 칸 비율에 맞춰 그릴 것.
- **버튼 PNG는 반드시 투명 배경(RGBA)** 으로 제작할 것. `background` 이미지 위에 버튼 아이콘이 겹쳐 그려지므로, 배경이 불투명하면 타이머 배경 디자인을 가린다.

#### 세로 구분선을 아이콘에 합쳐 넣기 (권장)

`buttonStyle: "skin"`일 때 세로선을 버튼 PNG 왼쪽 끝에 포함하면, 코드 구분선 없이 디자인 자유도를 최대로 가져갈 수 있다(그라데이션·커스텀 스타일 등 모두 가능).

```
[  −  ][│ ▶  ][│ +  ]   ← IDLE:     첫 칸 제외 나머지에 왼쪽 선 포함
[  ‖  ][│ ■  ]           ← RUNNING:  동일
```

이 방식을 쓸 때는 반드시 `showDividers: false`로 코드 세로선을 끌 것. 겹치면 선이 두 줄이 된다.

> 단, 첫 번째 칸(상태마다 다름 — IDLE: `−`, RUNNING/PAUSED: 재생·정지)에는 왼쪽 선이 없어야 자연스럽다.

## 4. RemoteViews 제약 (위젯이라 반드시 지킬 것)

위젯은 RemoteViews라 일반 View와 다르게 **제약**이 있다. 어기면 런처에 **"Can't load widget"**.

1. **순수 `<View>` 태그 금지** — 인플레이션 화이트리스트에 없음. 구분선/칸막이는 `TextView`/`ImageView`/`Space`로.
2. **비-remotable 메서드 호출 금지** — `setString/setInt` 등 리플렉션 호출은 `@RemotableViewMethod`만 가능.
   - 가능: `setTextColor`, `setTextViewTextSize` → **글자 색/크기는 직접 지정 OK**.
   - 불가: `setFontFamily(String)` → **글꼴(family)·커스텀 .ttf는 직접 불가**.
     커스텀 글꼴은 숫자를 `Typeface`로 **비트맵 렌더링 후 `setImageViewBitmap`** 방식으로만 가능(향후 확장 지점).
3. 비트맵은 IPC 크기 제한이 있으니 버튼/배경 PNG는 표시 크기에 맞게(과대 해상도 X).

진단: 위젯이 깨지면 에뮬레이터 재설치 후 `adb logcat | grep AppWidgetHostView` 로 정확한 예외/줄번호 확인.

## 5. "희끄무리한" 배경(런처 위젯 패널)

위젯 콘텐츠가 셀을 꽉 채우지 않고 투명 영역이 크면, 일부 런처(예: 삼성 One UI 등)는
위젯 뒤에 **옅은 둥근 배경 패널**을 깔아준다. 이는 타이머 스킨/크기 문제가 아니라 **런처 동작**이다.
(Pixel 런처 기준 테스트기에서는 타이머 주변이 깨끗하게 렌더링됨.)

완화 방법:
- 스킨이 영역 배경을 불투명/반투명으로 채워 "패널처럼" 보이게 디자인.
- 또는 그대로 두고 투명을 유지(런처별 차이로 수용).

> 캐릭터 프레임의 바닥 기준선 정렬, 캔버스 크기, 애니메이션 속도(`frameDurationMs`) 등
> **그림 제작 규칙은 스킨 리포 README로 이관**되었다.

## 6. 노스킨 규칙 요약

- 배경 투명, 박스 없음, 구분선 없음, 버튼 그림 없음. **`timer` 블록에 `showBox:false`,
  `showDividerH:false`, `showDividers:false`, `buttonStyle:"none"`을 명시**해야 한다.
  (※ `timer` 블록 자체를 생략하면 노스킨이 아니라 **기본 스킨**이 적용된다.)
- 단, **버튼 탭 영역(고정 크기 ImageView)은 유지** → 조작은 그대로 가능.
- 숫자 글꼴은 레이아웃 기본값. (어두운 배경에서 가독성이 필요하면 스킨에서 `font.color` 지정 권장.)
