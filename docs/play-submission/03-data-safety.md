# Data safety(데이터 보안) 설문 답변 초안

Play Console → **앱 콘텐츠 → 데이터 보안**. 코드 기준 실제 흐름에 맞춘 답변.

---

## 데이터 수집/공유 여부
- 앱이 사용자 데이터를 수집/공유합니까? → **예** (아래 항목만 한정)
- 모든 전송은 암호화됩니까? → **예 (HTTPS)**
- 사용자가 데이터 삭제를 요청할 수 있습니까? → 기기 내 데이터는 앱 삭제로 제거.
  서버 보관 영구 데이터 없음(구매 토큰은 검증 시점에만 사용). 문의 이메일 제공.

## 수집 데이터 항목

### 1) 금융 정보 → "구매 내역(Purchase history)"
- 수집함: **예**
- 목적: **앱 기능**(구매한 테마/이용권 잠금 해제)
- 사용자 식별과 연결됨? **아니요**(구매 검증 토큰만, 계정 프로필과 결합 안 함)
- 제3자와 공유? **아니요**(검증 인프라 Cloudflare는 처리 위탁이며 판매/광고 공유 아님)
- 선택적/필수? 구매 시에만 발생(선택적 기능)

### 2) 앱 활동/성능 → "진단(Diagnostics)" 또는 "크래시 로그"
- 수집함: **예** (카탈로그 로드 실패 시 오류 텍스트)
- 목적: **앱 기능/문제 해결**
- 사용자 식별과 연결됨? **아니요**(개인정보·기기 ID 없음)
- 제3자와 공유? **아니요**(개발자 알림 중계용)

## 수집하지 않는다고 명시할 항목 (전부 "아니요")
이름, 이메일, 사용자 ID, 주소, 전화번호, 위치(대략/정확), 연락처, 사진/동영상,
음성, 캘린더, 앱 목록, 웹 검색기록, **기기/광고 식별자**, 건강, 메시지.

## 보안 관행
- 전송 중 암호화: **예**
- 데이터 삭제 요청 경로 제공: **예**(문의 이메일)
- 독립 보안 검토(선택): 해당 없음

---

> 핵심 포인트: 광고/분석 SDK가 없고 개인 식별 데이터를 모으지 않으므로, 대부분
> 항목이 "아니요"입니다. 실제로 나가는 건 **구매 토큰(기능용)** 과 **진단 오류
> 텍스트(PII 없음)** 둘뿐입니다.
